# E2Eテスト仕様書

**作成日:** 2026-03-04

---

## 1. テスト環境

- テストフレームワーク: Playwright
- ブラウザ: Chromium, Firefox
- テスト対象URL: http://localhost:3000

---

## 2. テストシナリオ一覧

### 2.1 認証テスト

#### E2E-AUTH-01: 管理者ログイン成功
```
前提条件: 管理者アカウントが存在する（admin@example.com / password123）
手順:
  1. /login にアクセス
  2. メールアドレスに "admin@example.com" を入力
  3. パスワードに "password123" を入力
  4. 「ログイン」ボタンをクリック
期待結果:
  - /admin/dashboard にリダイレクトされる
  - ナビゲーションに管理者メニューが表示される
```

#### E2E-AUTH-02: 従業員ログイン成功
```
前提条件: 従業員アカウントが存在する（employee@example.com / password123）
手順:
  1. /login にアクセス
  2. メールアドレスに "employee@example.com" を入力
  3. パスワードに "password123" を入力
  4. 「ログイン」ボタンをクリック
期待結果:
  - /shifts にリダイレクトされる
  - シフト確認画面が表示される
```

#### E2E-AUTH-03: ログイン失敗（パスワード誤り）
```
手順:
  1. /login にアクセス
  2. 存在するメールアドレスを入力
  3. 誤ったパスワードを入力
  4. 「ログイン」ボタンをクリック
期待結果:
  - /login に留まる
  - エラーメッセージ「メールアドレスまたはパスワードが正しくありません」が表示される
```

#### E2E-AUTH-04: ログアウト
```
前提条件: ログイン済み
手順:
  1. 「ログアウト」ボタンをクリック
期待結果:
  - /login にリダイレクトされる
  - 再度 /admin/dashboard にアクセスすると /login にリダイレクトされる
```

#### E2E-AUTH-05: 未認証アクセスのリダイレクト
```
手順:
  1. 未ログイン状態で /admin/dashboard にアクセス
期待結果:
  - /login にリダイレクトされる
```

### 2.2 従業員管理テスト

#### E2E-EMP-01: 従業員追加
```
前提条件: 管理者でログイン済み
手順:
  1. 従業員一覧画面に移動
  2. 「従業員を追加」ボタンをクリック
  3. 名前: 「テスト 従業員」を入力
  4. メールアドレス: 「test_emp@example.com」を入力
  5. パスワード: 「password123」を入力
  6. 役割: 「従業員」を選択
  7. 「保存」ボタンをクリック
期待結果:
  - モーダルが閉じる
  - 一覧に「テスト 従業員」が追加される
```

#### E2E-EMP-02: 従業員編集
```
前提条件: 管理者でログイン済み、従業員が1件以上存在
手順:
  1. 従業員一覧画面で「編集」ボタンをクリック
  2. 名前を変更する
  3. 「保存」ボタンをクリック
期待結果:
  - モーダルが閉じる
  - 一覧の名前が更新される
```

#### E2E-EMP-03: 従業員削除
```
前提条件: 管理者でログイン済み、削除対象従業員が存在
手順:
  1. 従業員一覧で「削除」ボタンをクリック
  2. 確認ダイアログで「OK」をクリック
期待結果:
  - 確認ダイアログが閉じる
  - 一覧から対象従業員が削除される
```

### 2.3 シフト管理テスト

#### E2E-SHIFT-01: シフト作成
```
前提条件: 管理者でログイン済み、従業員が1件以上存在
手順:
  1. シフト作成画面に移動
  2. カレンダーの空の日付セルをクリック
  3. 従業員を選択
  4. 開始時刻: 09:00、終了時刻: 18:00 を入力
  5. 「保存」ボタンをクリック
期待結果:
  - モーダルが閉じる
  - カレンダーに従業員名と時刻が表示される
```

#### E2E-SHIFT-02: シフト編集
```
前提条件: シフトが1件以上存在
手順:
  1. カレンダーのシフトチップをクリック
  2. 開始時刻を変更
  3. 「保存」をクリック
期待結果:
  - カレンダーの表示が更新される
```

#### E2E-SHIFT-03: シフト削除
```
手順:
  1. カレンダーのシフトチップをクリック
  2. 「削除」ボタンをクリック
  3. 確認ダイアログで「OK」をクリック
期待結果:
  - カレンダーからシフトが削除される
```

#### E2E-SHIFT-04: シフト確定
```
前提条件: 未確定シフトが存在
手順:
  1. 「シフト確定」ボタンをクリック
  2. 確認ダイアログで「OK」をクリック
期待結果:
  - 全シフトのステータスが「確定済み」になる
```

#### E2E-SHIFT-05: CSVエクスポート
```
前提条件: シフトが存在
手順:
  1. 「CSVエクスポート」ボタンをクリック
期待結果:
  - CSVファイルがダウンロードされる
  - ファイル名が shift_{year}{month}.csv 形式である
```

### 2.4 希望シフトテスト

#### E2E-REQ-01: 希望シフト入力
```
前提条件: 従業員でログイン済み
手順:
  1. 希望シフト入力画面に移動
  2. 翌月のカレンダーの日付をクリック
  3. 「希望あり」を選択
  4. 開始時刻: 09:00、終了時刻: 18:00 を入力
  5. 「保存」をクリック
期待結果:
  - カレンダーに「○」マークが表示される
```

#### E2E-REQ-02: 希望シフト修正
```
前提条件: 希望シフトが1件存在（未確定月）
手順:
  1. 入力済みの日付セルをクリック
  2. 「不可」に変更
  3. 「保存」をクリック
期待結果:
  - カレンダーの表示が「×」に変更される
```

### 2.5 シフト確認テスト

#### E2E-VIEW-01: 自分のシフト確認
```
前提条件: 従業員でログイン済み、確定済みシフトが存在
手順:
  1. シフト確認画面に移動
期待結果:
  - カレンダーに自分のシフト（時刻）が表示される
  - 勤務日数と合計勤務時間が正しく表示される
```

#### E2E-VIEW-02: 月切り替え
```
手順:
  1. 「次月」ボタンをクリック
期待結果:
  - 翌月のカレンダーが表示される
```

#### E2E-VIEW-03: スマートフォン表示確認
```
手順:
  1. ブラウザを375px幅に縮小
  2. シフト確認画面を表示
期待結果:
  - レスポンシブレイアウトが適用される
  - コンテンツが正常に表示される
```
