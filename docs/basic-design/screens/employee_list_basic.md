# 従業員一覧画面 基本設計書

**画面ID:** SCR-04
**画面名:** 従業員一覧画面
**対象ユーザー:** 管理者
**作成日:** 2026-03-04

---

## 1. 画面概要

管理者が従業員の登録・編集・削除を行う画面。登録済み従業員の一覧を表示し、各種操作が可能。

---

## 2. 画面レイアウト

```
+----------------------------------------------------------+
| シフト管理システム              管理者: [名前] [ログアウト]  |
+----------------------------------------------------------+
| [シフト管理] [従業員管理] [希望シフト確認]                  |
+----------------------------------------------------------+
|  従業員一覧                          [+ 従業員を追加]      |
+----------------------------------------------------------+
|  名前          | メール              | 役割  | 操作        |
|----------------|---------------------|-------|------------|
|  田中 太郎     | tanaka@example.com  | 従業員 | [編集][削除] |
|  鈴木 花子     | suzuki@example.com  | 管理者 | [編集][削除] |
|  佐藤 次郎     | sato@example.com    | 従業員 | [編集][削除] |
+----------------------------------------------------------+
```

---

## 3. 機能一覧

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SCR-04-F01 | 従業員一覧表示 | 登録済み従業員の一覧表示 |
| SCR-04-F02 | 従業員追加 | 新規従業員の追加モーダル表示 |
| SCR-04-F03 | 従業員編集 | 既存従業員情報の編集モーダル表示 |
| SCR-04-F04 | 従業員削除 | 従業員の削除（確認ダイアログ付き） |

---

## 4. 従業員追加・編集モーダル

```
+----------------------------------+
|  従業員追加/編集                   |
|                                  |
|  名前: [___________________]     |
|  メールアドレス: [____________]   |
|  パスワード: [_______________]    |
|  役割: [管理者 / 従業員]          |
|                                  |
|  [保存] [キャンセル]               |
+----------------------------------+
```

---

## 5. 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|------|--------------|
| 名前 | text | 必須 | 最大50文字 |
| メールアドレス | text | 必須 | メール形式、重複不可 |
| パスワード | password | 必須（新規のみ） | 最低8文字 |
| 役割 | select | 必須 | admin / employee |

---

## 6. 画面遷移

| 条件 | 遷移先 |
|------|--------|
| 削除確認ダイアログ「OK」 | 同画面（一覧更新） |

---

## 7. 関連API

| API | 用途 |
|-----|------|
| GET /api/employees | 従業員一覧取得 |
| POST /api/employees | 従業員新規登録 |
| PUT /api/employees/:id | 従業員情報更新 |
| DELETE /api/employees/:id | 従業員削除 |
